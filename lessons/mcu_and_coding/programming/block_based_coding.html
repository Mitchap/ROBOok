First Words: Block-Based Coding in Tinkercad

Ready to start speaking robot? One of the easiest ways to begin programming is with block-based coding. Imagine snapping together colorful LEGO blocks, but instead of building a tower, you're building a set of instructions for a virtual robot! Tinkercad provides a fantastic way to learn this, and in this lesson, we'll take our first steps into the world of programming using Tinkercad's visual block interface.

What is Block-Based Programming? Coding with Digital LEGOs üß±üíª

Imagine building something cool with LEGO blocks. Each block has a specific shape and a specific thing it can do, right? You snap them together in a certain order to create a car, a spaceship, or whatever you can dream up.

Block-based programming is super similar! It's a visual way to write code by dragging and dropping pre-made blocks instead of typing lines of text. Each block is like a digital LEGO brick that represents a specific programming command or function ‚Äì telling the computer to do something specific. You connect these blocks in a particular order to create a sequence of instructions, just like building with LEGOs!

Common Programming Blocks

Introduce some of the most common blocks:

  • "wait" block: Pauses the program for a specified time.
  • "set pin" block: Sets a digital pin to HIGH (on) or LOW (off).
  • "read digital pin" block: Reads the value of a digital input pin.
  • "read analog pin" block: Reads the value of an analog input pin.
  • "repeat" block: Executes a set of blocks multiple times (loop).
  • "if...do" block: Executes a set of blocks only if a condition is true (conditional statement).

Explain the function of each block with simple examples.

Basic Programming Concepts with Blocks

Explain how block-based coding helps to understand fundamental programming concepts:

  • Sequence: The order in which the blocks are arranged determines the order in which the instructions are executed.
  • Loops: Using "repeat" blocks to execute a set of instructions multiple times.
  • Conditional statements: Using "if...do" blocks to make decisions based on conditions.

Creating Simple Programs with Blocks

Provide step-by-step examples of how to create simple programs using Tinkercad blocks:

Example 1: Blinking an LED:

  • Use "set pin" blocks to turn the LED on and off.
  • Use "wait" blocks to control the timing.
  • Use a "repeat" block to make the LED blink continuously.

Example 2: Reading a virtual button and turning on an LED:

  • Use "read digital pin" to read the button state.
  • Use an "if...do" block to check if the button is pressed.
  • Use a "set pin" block to turn the LED on if the button is pressed.

Block Categories Overview

Here's a quick look at some of the block categories you'll find:

  • Control: Blocks that manage the flow of your program (like waiting, repeating, or making decisions).
  • Input: Blocks that get information from sensors or the virtual environment (like reading button presses or sensor values).
  • Output: Blocks that control things like turning LEDs on/off or setting motor speeds.
  • Math: Blocks for performing calculations.

Common Programming Blocks üí™

Let's check out some of the blocks you'll use a lot:

wait (1) seconds: This block tells the program to pause for the specified amount of time (in seconds). Think of it as the robot taking a little break before doing the next thing.

Example: wait (0.5) seconds makes the program pause for half a second.

set pin (13) to HIGH / set pin (13) to LOW: This block controls the digital output pins of the Arduino. HIGH usually means turning something ON (like sending power), and LOW means turning it OFF (stopping the power). The number in the block specifies which pin to control.

Example: set pin (8) to HIGH turns on whatever is connected to digital pin 8.

read digital pin (2): This block reads the state (HIGH or LOW) of a specific digital input pin. It's like asking if a button connected to that pin is pressed (HIGH) or not (LOW).

Example: If a button is connected to digital pin 2, this block will tell you if it's currently being pressed.

read analog pin (A0): This block reads the voltage value from a specific analog input pin. Analog pins can read a range of values, like the brightness from a light sensor. The value you get will be a number representing that voltage.

Example: If a light sensor is connected to analog pin A0, this block will give you a number that tells you how bright it is.

repeat (10) times: This block is a loop. It tells the program to execute the blocks placed inside it a specific number of times. It's great for making things happen repeatedly.

Example: Putting set pin (13) to HIGH inside a repeat (3) times block will turn pin 13 ON three times in a row (very quickly!).

if (condition) do: This is a conditional statement. The blocks inside this block will only be executed if the condition you specify is true. It allows your program to make decisions.

Example: if (read digital pin (7) == HIGH) do { set pin (12) to HIGH } means "IF the button on digital pin 7 is pressed (HIGH), THEN turn ON the LED connected to digital pin 12."

Basic Programming Concepts with Blocks üß†

Block-based coding is a fantastic way to grasp fundamental programming ideas without getting bogged down in typing rules:

  • Sequence: The order you snap the blocks together is crucial! The Arduino will execute the instructions in that exact order, one after the other, from top to bottom. It's like following the steps in a recipe.
  • Loops: The repeat block shows you how to make the Arduino do the same set of instructions over and over again. This is super useful for things like blinking an LED continuously or making a robot move in a circle.
  • Conditional statements: The if...do block introduces the idea of decision-making. Your program can check if something is true or false and then do different things based on the result. This is how robots can react to their environment.

Creating Simple Programs with Blocks üöÄ

Let's build a couple of simple programs using Tinkercad blocks:

Example 1: Blinking an LED

  1. Make sure you have an Arduino and an LED connected with a resistor in your Tinkercad circuit (like we did in the previous lesson). Let's say the LED is connected to digital pin 13.
  2. Open the Code editor and make sure you're in Blocks mode.
  3. Drag out the following blocks and connect them:
    4. 
  [repeat [forever]]
  [set pin [13] to [HIGH]]
  [wait [1] seconds]
  [set pin [13] to [LOW]]
  [wait [1] seconds]
  4. Click "Start Simulation". You should see your virtual LED blinking on and off every second!

Example 2: Reading a Virtual Button and Turning on an LED

  1. In Tinkercad Circuits, add a Button component to your circuit and connect one side of it to a digital input pin on the Arduino (e.g., pin 7). Connect the other side of the button to 5V (through a pull-up or pull-down resistor for proper behavior, but Tinkercad often handles this simply). Also, make sure you have an LED connected to a digital output pin (e.g., pin 12) with a resistor.
  2. Open the Code editor in Blocks mode.
  3. Drag out and connect the following blocks: 

[repeat [forever]]
[if [ [read digital pin [7]] == [HIGH] ]]
[set pin [12] to [HIGH]]
[else]
[set pin [12] to [LOW]]
[end if]
  1. Click "Start Simulation". Now, when you virtually "click" the button in Tinkercad, the LED should turn on. When you release the button, the LED should turn off.

These simple examples show how you can use block-based programming to create interactive circuits and control virtual components in Tinkercad! As you get more comfortable with these blocks, you'll be able to build much more complex and interesting programs for your virtual robots.
