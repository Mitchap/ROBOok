The Language of Robots: How Microcontrollers Understand Instructions

Imagine trying to teach a dog a new trick. You can't just speak to it in English; you need to use specific commands and actions it understands. Similarly, robots don't understand human languages. Instead, we use special languages called 'programming languages' to give them instructions. In this lesson, we'll explore the fundamental concept of programming and how we use these languages to communicate with robots.

  • Define a programming language.
  • Explain the purpose of programming languages in robotics.
  • Introduce the concept of giving instructions to a computer.
  • Distinguish between different types of programming languages (briefly).

What is Programming? Telling Computers What to Do ü§ñüíª

Imagine you want to bake a cake üéÇ. You can't just throw ingredients together and hope for the best, right? You need a recipe! This recipe is a set of step-by-step instructions that tells you exactly what to do: mix flour, add sugar, bake at a certain temperature, etc.

Programming is kind of like writing a recipe, but instead of telling a baker what to do, you're creating a set of instructions that a computer (or a robot!) can follow. The recipe is the "program," and the computer or robot is the "baker."

So, in simple terms, programming is how we tell computers (and robots) what to do!

Programming Languages: Talking to Machines üó£Ô∏è

Computers and robots don't understand English or Tagalog the way we do. They need a special way of being told what to do, and that's where programming languages come in.

A programming language is a formal language with its own specific rules and syntax (like grammar in a human language). It's like a secret code we use to write computer programs. Different programming languages are designed for different purposes, just like there are different kinds of recipes for different kinds of food.

There are tons of different programming languages out there! You might have heard of some like Python, Java, or C++. In this course, we're going to focus on the Arduino programming language, which is great for controlling electronics and robots.

Why Programming for Robots? ‚öôÔ∏èüß†

Why do robots need programming? Well, they're not born knowing what to do! Robots need instructions to perform tasks. Programming is the key to giving them those instructions.

    With programming, we can:

  • Control a robot's behavior: Make it move forward, turn, grab things, or even dance!
  • Enable decision-making: Program robots to react to their surroundings based on information from sensors.
  • Make them interact with their environment: Allow them to respond to light, sound, touch, and other inputs.
  • Achieve specific goals: Program robots to complete tasks, whether it's navigating a maze or assembling a product.

Basically, programming is what makes a robot smart and allows it to do useful things!

Giving Instructions to a Computer üìù‚û°Ô∏èüíª

The basic process of writing a program goes something like this:

  1. Break it down: First, you need to figure out the task you want the computer or robot to do and break it down into smaller, logical steps.
  2. Write the steps: Then, you write those steps down using the specific rules of a programming language.
  3. Execute: The computer or robot reads these instructions one by one, in order, and performs them.

Let's take a super simple example: making an LED turn on. We can think of the steps like this:


  START
      Turn the LED ON
  END

In a programming language (like Arduino), this might look something like this (we'll learn this later!):

  
  Arduino
  
    digitalWrite(ledPin, HIGH); // Turn the LED ON

Types of Programming Languages (Quick Look) üëÄ

Just like there are different ways to communicate with people (talking, writing, sign language), there are different types of programming languages. Two main types you'll encounter are:

  • Text-based languages: These are like writing instructions using words, symbols, and specific grammar. Arduino, C++, and Python are examples of text-based languages. You type out lines of code.
  • Block-based languages: These are more visual. Instead of typing lines of code, you drag and drop colorful blocks that represent different commands and connect them like puzzle pieces. Scratch (which you might have seen before) and the block-based coding in Tinkercad are examples of this.

In this course, we're going to start with block-based coding in Tinkercad because it's a really fun and easy way to learn the basics of programming. Once you're comfortable with the concepts, we'll move on to text-based coding with Arduino, which gives you even more control over your robots and projects!
