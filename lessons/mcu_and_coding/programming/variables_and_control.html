📘 Lesson 4

From Blocks to Text: Introducing Arduino Code

To make robots truly useful, we need to give them the ability to make decisions and repeat actions. This is where variables and control structures come in. Variables are like containers that hold information, and control structures are the rules that determine how a program flows. In this lesson, we'll explore these fundamental programming concepts using both Tinkercad blocks and Arduino text code, and see how they empower robots to perform complex tasks.

🎯 What You'll Learn in This Lesson:

  • Define variables and explain their purpose in programming.
  • Introduce fundamental control structures:
    • Conditional statements (if/else)
    • Loops (for, while)
  • Implement variables and control structures using Tinkercad blocks.
  • Implement variables and control structures using Arduino text code.
  • Understand how variables and control structures enable robots to perform more complex behaviors.

Variables: Your Program's Memory Boxes 📦

  • Imagine you have a bunch of labeled boxes. Each box has a name tag on it, like "score," "temperature," or "robotSpeed." Inside these boxes, you can put different items – numbers, words, or even true/false values.
  • In programming, these labeled boxes are called variables. A variable is a named storage location in the computer's memory that can hold a value. The label on the box is the variable name, and the items inside are the values the variable holds.
  • Why are variables important?
    • Storing and Manipulating Data: Variables let your program remember information and change it as needed. For example, you can store the distance read by a sensor in a variable and then use that value to decide what the robot should do.
    • Flexibility and Reusability: Instead of using specific values directly in your code, you can use variables. This makes your programs more flexible. If you need to change a value (like the speed of a motor), you only need to change it in the variable, not everywhere it's used in your code. This makes your code easier to update and reuse for different situations.
  • Using Variables in Tinkercad Blocks:
    • In the Tinkercad Code editor (Blocks mode), click on the "Variables" category.
    • Click the "Create Variable..." button and give your variable a name (e.g., speed, buttonState).
    • You'll now see blocks like:
    • set [variable name] to [value]: This block is like putting an item into your labeled box. It assigns a specific value to your variable name.
    • change [variable name] by [value]: This block is like adding or removing items from your box. It increases or decreases the current value of your variable name by the specified value.
    • You can also use the variable's name block (e.g., speed) to access the value stored in it within other blocks (like in math calculations or comparisons).
    • (Imagine a screenshot of the Tinkercad Variables blocks category with "set variable" and "change variable" highlighted)
  • Using Variables in Arduino Code:
    • In Arduino (text-based), you need to declare a variable before you can use it. This means telling the computer the variable's name and what type of data it will hold. We use data types to do this:
    • int (integer): For whole numbers (e.g., -10, 0, 5).
    • float (floating-point): For numbers with decimal points (e.g., 3.14, -0.5).
    • String: For sequences of characters (text) (e.g., "Hello", "Robot Name").
    • boolean (bool): For true or false values.

To declare a variable, you write the data type, followed by the variable name, and then you can optionally assign an initial value using the = operator:

    int score = 0;     // Declares an integer variable named 'score' and sets it to 0

    float temperature; // Declares a floating-point variable named 'temperature' (no initial value)

    String message = "Starting"; // Declares a String variable and sets it to "Starting"

    boolean isButtonPressed = false; // Declares a boolean variable and sets it to false

To change the value of a variable later in your code, you simply use the variable name and the = operator:

    score = 10;          // Changes the value of 'score' to 10

    temperature = 25.5;  // Changes the value of 'temperature' to 25.5

    message = "Running"; // Changes the value of 'message' to "Running"

    isButtonPressed = true; // Changes the value of 'isButtonPressed' to true

Conditional Statements: Making Decisions 🤔

Conditional statements are like asking a question in your program. They allow your program to execute different blocks of code depending on whether a certain condition is true or false. This is how you make your robot behave differently in different situations.

The most common conditional statements are if and if...else.

Logic Flowchart: 

            START
              |
              V
          +-------+
          | Check | <--- Condition (True or False?)
          | Condition |
          +-------+
          /       \
         True     False
          |         |
          V         V
    +-------+     +-------+
    | Do    |     | Do    | <--- Code Block (Executed if False)
    | This  |     | That  |
    | Code  |     +-------+
    +-------+
      |
      V
    END (or Continue)
  • if statement: If the condition is true, the code inside the if block is executed. If the condition is false, the code inside the if block is skipped.
  • if...else statement: If the condition is true, the code inside the if block is executed. If the condition is false, the code inside the else block is executed.

📘 Conditional Statements in Tinkercad Blocks

In the Tinkercad Code editor (Blocks mode), click on the "Control" category.

  • You'll find the if [condition] do and if [condition] do { ... } else { ... } blocks.
  • The [condition] part is where you place blocks that evaluate to true or false (usually using blocks from the "Math" or "Input" categories to compare values).
  • The blocks you want to execute based on the condition go inside the do (or the if and else sections).

(Imagine a screenshot of the Tinkercad Control blocks category with "if...do" and "if...else" highlighted)

📗 Conditional Statements in Arduino Code

if (condition) {
          // Code to execute if the condition is true
        }
        
        if (condition) {
          // Code to execute if the condition is true
        } else {
          // Code to execute if the condition is false
        }

The (condition) is an expression that evaluates to either true or false. We often use comparison operators to create these conditions:

  • == (equal to)
  • != (not equal to)
  • < (less than)
  • > (greater than)
  • <= (less than or equal to)
  • >= (greater than or equal to)

Examples:

int temperature = 22;
        if (temperature > 25) {
          // Turn on the fan
        }
        
        int buttonState = digitalRead(7);
        if (buttonState == HIGH) {
          // Robot moves forward
        } else {
          // Robot stops
        }

🔄 Loops: Repeating Actions

Loops are a way to make your program execute a block of code repeatedly without having to write the same instructions over and over again. This is super useful for tasks you want your robot to do multiple times.

The most common types of loops are for and while.

Loop Diagram (General Concept):

(Diagram Description)

  • START → Initialize Counter/Condition → Check Condition
  • → If True: Do Code → Update Counter → (Loop)
  • → If False: Exit

for loop

Used when you know how many times you want to repeat a block of code. It usually involves a counter that starts at a certain value, increases (or decreases) each time the loop runs, and stops when it reaches a certain condition.

while loop

Used when you want to repeat a block of code as long as a certain condition is true. The loop continues to run until the condition becomes false.

🧱 Loops in Tinkercad Blocks

In the Tinkercad Code editor (Blocks mode), click on the "Control" category.

  • The repeat [number] times block is a simple form of a for loop.
  • You can use the repeat until [condition] block for while-like behavior.

(Imagine a screenshot of the Tinkercad Control blocks category with "repeat" and "repeat until" highlighted)

💻 Loops in Arduino Code

for loop:

for (initialization; condition; increment/decrement) {
          // Code to be repeated
        }
  • initialization: Sets up a counter variable (runs once at the beginning).
  • condition: Checked before each iteration. The loop continues as long as it's true.
  • increment/decrement: Changes the counter variable after each iteration.

Example:

for (int i = 0; i < 5; i++) { // Repeat 5 times
          digitalWrite(13, HIGH);
          delay(500);
          digitalWrite(13, LOW);
          delay(500);
        }

while loop:

while (condition) {
          // Code to be repeated as long as the condition is true
          // Make sure something inside the loop changes the condition eventually!
        }
  • condition: Checked before each iteration. It's crucial to have something inside the loop that will eventually make the condition false.

Example:

int distance = 100;
        while (distance > 50) {
          // Move forward
          delay(100);
          distance = readDistanceSensor(); // Assume this function reads the distance
        }
        // Stop moving

🤖🧠 Using Variables and Control Structures in Robot Programming

Example 1: Robot Moves Until Obstacle

// Assume 'distanceSensorPin' is connected to a distance sensor
        int obstacleDistance = 30; // Threshold distance in cm
        
        void loop() {
          int currentDistance = analogRead(distanceSensorPin); // Read distance
        
          while (currentDistance > obstacleDistance) {
            // Move robot forward (control motors here)
            delay(50);
            currentDistance = analogRead(distanceSensorPin); // Keep updating distance
          }
          // Stop robot (stop motors here)
        }

(Tinkercad Blocks equivalent would involve a 'while' loop using the 'repeat until' block with a comparison of the analog sensor reading to a value.)

Example 2: Robot Turns Based on Button Press

int leftButtonPin = 2;
        int rightButtonPin = 3;
        int leftMotorPin = 9;
        int rightMotorPin = 10;
        
        void setup() {
          pinMode(leftButtonPin, INPUT_PULLUP);
          pinMode(rightButtonPin, INPUT_PULLUP);
          pinMode(leftMotorPin, OUTPUT);
          pinMode(rightMotorPin, OUTPUT);
        }
        
        void loop() {
          if (digitalRead(leftButtonPin) == LOW) {
            digitalWrite(leftMotorPin, HIGH);
            digitalWrite(rightMotorPin, LOW);
            delay(500);
            digitalWrite(leftMotorPin, LOW);
            digitalWrite(rightMotorPin, LOW);
          } else if (digitalRead(rightButtonPin) == LOW) {
            digitalWrite(leftMotorPin, LOW);
            digitalWrite(rightMotorPin, HIGH);
            delay(500);
            digitalWrite(leftMotorPin, LOW);
            digitalWrite(rightMotorPin, LOW);
          } else {
            // Move robot forward (or do nothing)
          }
          delay(100);
        }

(Tinkercad Blocks equivalent would use 'if...else if...else' blocks with digital input readings and 'set pin' blocks for motor control.)

Example 3: Robot Follows Steps

int steps = 4;
        int forwardTime = 1000; // Milliseconds
        
        void setup() {
          // Configure motor control pins as OUTPUTs
        }
        
        void loop() {
          for (int i = 0; i < steps; i++) {
            // Move robot forward
            delay(forwardTime);
            // Stop robot
            delay(500);
          }
          delay(5000); // Wait before repeating
        }

(Tinkercad Blocks equivalent would use a 'repeat [steps] times' block with blocks for motor control and delays inside.)

📝 Summary and Key Takeaways

Block-based programming is like building with digital LEGO bricks 🧱💡—each block is a command that tells the robot what to do. Instead of writing code, you drag and snap these visual blocks together to create programs. It’s fun, easy, and a perfect way to start your coding journey!

  • Variables are like memory boxes 📦 that store information your program needs to use and change, such as speed, distance, or status.
  • Conditionals (like if and if...else) allow your robot to make decisions based on sensor input or data—just like choosing what to do based on the weather!
  • Loops (like for and while) let your robot repeat actions without writing code over and over—perfect for tasks that happen many times.
  • In Tinkercad Blocks, variables and control structures are visual and beginner-friendly. You can easily set, change, and use conditions and loops.
  • In Arduino Code, you declare variables with data types (int, float, boolean, etc.) and write control structures in a structured syntax.
  • Together, these tools let you build smarter, more responsive robots that interact with their environment and follow sequences—just like a mini helper with a plan! 🤖
